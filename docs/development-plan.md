## 거래 모드

| 모드 | 설명 |
|------|------|
| 현물 (Spot) | USDT 마켓, 포지션 평균 단가 합산 |
| 선물 (Futures) | USDT-M 무기한 계약, 레버리지 1x~125x, 롱/숏 |

### 잔고 구분
- 현물과 선물은 **각각 별도의 USDT 잔고**를 보유
- 현물 잔고: 초기 **100 USDT**
- 선물 잔고: 초기 **0 USDT**
- **현물 ↔ 선물 이체 기능** 제공 (양방향)

### 포지션 관리 규칙

**현물**
- 동일 코인 여러 번 매수 시 평균 단가로 합산 (단일 포지션)
- 표시 방식: 진입가(평균 매수가), 보유 수량, 미실현 PnL, ROI 포지션 형태로 표시
- 부분 청산 가능 (지정 수량만 매도)

**선물**
- 포지션 방향: **헤지 모드** — 동일 코인에서 롱/숏 동시 보유 가능
- 동일 코인 + 동일 방향 + 동일 레버리지: 평균 단가 합산
- 동일 코인 + 동일 방향 + 다른 레버리지: 별개 포지션
- 동일 코인 + 다른 방향 (롱/숏): 항상 별개 포지션 (반대 방향 진입 시 기존 포지션 청산 없이 새 포지션 생성)
- 부분 청산 가능 (지정 수량만 청산)

## 선물 거래 사양

| 항목 | 내용 |
|------|------|
| 계약 종류 | USDT-M 무기한 계약 (Perpetual) |
| 레버리지 | 1x ~ 125x |
| 마진 방식 | 격리 마진 (Isolated) / 교차 마진 (Cross) |
| 포지션 방향 | 롱 (Long) / 숏 (Short) |
| 펀딩비 | 8시간마다 부과 (바이낸스 실제 펀딩비율 적용) |
| 청산 | 유지 증거금 미달 시 강제 청산 시뮬레이션 |

### 주요 계산식
- **진입 증거금** = 포지션 크기 ÷ 레버리지
- **청산가 (롱)** = 진입가 × (1 − 1/레버리지 + 유지증거금률)
- **청산가 (숏)** = 진입가 × (1 + 1/레버리지 − 유지증거금률)
- **미실현 PnL (롱)** = (현재가 − 진입가) × 수량
- **미실현 PnL (숏)** = (진입가 − 현재가) × 수량
- **ROI** = 미실현 PnL ÷ 진입 증거금 × 100%
- **펀딩비** = 포지션 크기 × 펀딩비율 (양수: 롱→숏 지급, 음수: 숏→롱 지급)

- 표시 내용:
  - 총 자산 (현물 잔고 + 현물 보유 코인 평가금액 + 선물 잔고 + 선물 미실현 PnL)

    - 라인 차트: 시간대별 총 자산 변화
    - 바 차트: 일별 손익

- 탭: **USDT**

### 이체 모달
- 방향 선택 토글: **현물 → 선물** / **선물 → 현물**

- ROI(%) / PnL(USDT) 방식 선택 탭
- TP 가격 입력란
  - 청산 비율 슬라이더 (25% / 50% / 75% / 100% 프리셋)
  - 슬라이더 옆 실시간 청산 수량/금액 표시 (예: 50% → 5,000 USDT)
- SL 가격 입력란
  - 청산 비율 슬라이더 (25% / 50% / 75% / 100% 프리셋)
  - 슬라이더 옆 실시간 청산 수량/금액 표시 (USDT 기준)

### 트레이드 폼

- 지정가 / 시장가 탭
- 가격, 수량 입력
- 슬라이더 (25% / 50% / 75% / 100%)
- TP/SL 토글 섹션 (접었다 펼치기)

- 마진 방식 선택 (격리 / 교차, 토글 버튼)
- 롱 / 숏 버튼 (현물의 매수/매도 대신)
- 지정가 / 시장가 탭
- 가격, 수량 입력
- 슬라이더 (25% / 50% / 75% / 100%)
- TP/SL 토글 섹션 (접었다 펼치기)

### 수수료율 (바이낸스 기준)

| 구분 | Maker (지정가) | Taker (시장가) |
|------|--------------|--------------|

### 차감 방식
- 매수/매도 각각 1회씩 부과 (왕복 2회)
- **받는 자산에서 차감** (바이낸스 기본 방식)
  - 현물 매수: 받는 코인에서 차감 (예: BTC 매수 → BTC에서 0.1% 차감)
  - 현물 매도: 받는 USDT에서 차감 (예: BTC 매도 → USDT에서 0.1% 차감)
  - 선물: 받는 USDT(수익)에서 차감, 또는 증거금에서 차감

## 사용 API

### 현물 REST API (`api.binance.com`)
- `GET /api/v3/ticker/24hr` — 24시간 시세 (가격, 변동률, 거래량)
- `GET /api/v3/depth` — 호가창 스냅샷
- `GET /api/v3/trades` — 최근 체결 내역
- `GET /api/v3/klines` — 캔들 데이터

### 현물 WebSocket (`stream.binance.com`)
- `<symbol>@ticker` — 실시간 가격/24h 시세
- `<symbol>@depth20` — 실시간 호가창
- `<symbol>@trade` — 실시간 체결 내역
- `<symbol>@kline_<interval>` — 실시간 캔들

### 선물 REST API (`fapi.binance.com`)
- `GET /fapi/v1/ticker/24hr` — 선물 24시간 시세
- `GET /fapi/v1/depth` — 선물 호가창 스냅샷
- `GET /fapi/v1/trades` — 선물 체결 내역
- `GET /fapi/v1/klines` — 선물 캔들 데이터
- `GET /fapi/v1/premiumIndex` — 현재 펀딩비율 및 마크 가격
- `GET /fapi/v1/fundingRate` — 펀딩비 이력

### 선물 WebSocket (`fstream.binance.com`)
- `<symbol>@ticker` — 선물 실시간 시세
- `<symbol>@depth20` — 선물 실시간 호가창
- `<symbol>@trade` — 선물 실시간 체결 내역
- `<symbol>@kline_<interval>` — 선물 실시간 캔들
- `<symbol>@markPrice` — 마크 가격 + 펀딩비율 실시간

### Phase 6 — TP/SL (Take Profit / Stop Loss)
**목표:** 주문 시 익절/손절 조건 설정 및 자동 체결 시뮬레이션

#### 설정 방식 (2가지 중 선택)

| 방식 | 입력 | 트리거 가격 계산 |
|------|------|----------------|
| ROI (%) | 목표 수익률 % | 진입가 × (1 ± ROI%) |
| PnL (USDT) | 목표 손익 금액 | 진입가 ± (PnL ÷ 수량) |

#### 수량 비율 설정
- TP/SL 각각 **포지션의 몇 %를 청산**할지 설정 가능
- 예: TP 50% (절반만 익절), SL 100% (전량 손절)
- 슬라이더 또는 직접 입력 (25% / 50% / 75% / 100% 프리셋)

#### 동작 방식
- **주문 시 동시 설정**: 매수/매도 폼에서 주문과 함께 TP/SL 등록
- **포지션 후 추가/수정**: 체결된 포지션 목록에서 TP/SL 별도 설정 및 수정 가능
- 실시간 현재가가 트리거 가격 도달 시 자동 체결 시뮬레이션
- TP와 SL 동시 설정 가능 (먼저 도달한 쪽 체결 후 나머지 취소)

#### 구현 항목
- [ ] `js/tpsl.js` — TP/SL 로직 (설정, 모니터링, 자동 체결)
- [ ] ROI 기준 트리거 가격 계산
- [ ] PnL 기준 트리거 가격 계산 (USDT 기준)
- [ ] 청산 비율 슬라이더 (TP/SL 각각)
- [ ] 현재가 실시간 모니터링 → 트리거 도달 시 자동 체결
- [ ] TP/SL 설정값 `localStorage` 저장 (새로고침 후 유지)
- [ ] 보유 포지션 목록 UI에서 TP/SL 추가/수정/취소 가능
- [ ] 활성 TP/SL 목록 UI (설정된 조건 한눈에 확인)
- [ ] `css/tpsl.css` — TP/SL UI 스타일

**산출물:** `js/tpsl.js`, `css/tpsl.css`
